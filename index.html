Key Frameworks for AI PMs

Prioritization
RICE Score
Reach × Impact × Confidence ÷ Effort. Add "accuracy delta" as an impact multiplier for AI features.
Discovery
Opportunity Solution Tree
Teresa Torres' OST adapted for AI: desired outcome → opportunities → AI solutions → experiments.
Strategy
MECE Problem Tree
Decompose AI product challenges without overlap. Separate model quality, data, UX, and adoption branches.
Decision
Two-Way Door Test
Classify AI decisions as reversible (two-way) or irreversible (one-way). Move fast on two-way; slow on one-way.
Ethics
Consequence Scanning
Assess intended, unintended, and potential harmful consequences of AI features before shipping.
Build vs Buy
AI Stack Decision Matrix
Score: differentiation value, data sensitivity, team capability, time-to-market, and long-term cost.
Metrics
North Star Framework
One metric that captures value delivery. For AI: tasks completed with AI assistance, AI-enabled time saved.
UX
Uncertainty → Trust Ladder
Five levels: aware → try → trust → rely → advocate. Design AI features to move users up each rung.

Checklists
Work through these before shipping any AI feature or product.

🚦 Pre-Launch AI Feature

0 / 10 complete
  • Success criteria defined with measurable thresholds (precision, recall, or task completion rate)
  • Edge cases catalogued: low-confidence outputs, out-of-distribution inputs, adversarial inputs
  • Human fallback mechanism designed and tested for all failure modes
  • Eval set of ≥100 representative examples with expected outputs created
  • Bias and fairness audit completed across demographic dimensions
  • Confidence scores and uncertainty shown in UI where applicable
  • Data consent, privacy, and retention policy reviewed and documented
  • Model card written and shared with stakeholders
  • Rollback plan documented: feature flag ready, prior behavior preserved
  • Monitoring and alerting live: quality drift, error rate, cost-per-request

🧪 Experiment Design

0 / 8 complete
  • Clear hypothesis stated: "We believe X will cause Y for users who Z"
  • Primary metric and guardrail metrics defined before test starts
  • Sample size calculated for statistical significance at α=0.05, power=0.8
  • Minimum detectable effect (MDE) defined and business-validated
  • Holdout/control group isolation confirmed — no cross-contamination
  • Novelty effect accounted for (minimum 2-week run for behavioral tests)
  • Segment analysis plan defined: new vs returning, power vs casual users
  • Decision rule pre-committed: conditions to ship, hold, or kill the variant

⚖️ Responsible AI Review

0 / 8 complete
  • Consequence scan completed: intended, unintended, and harmful outcomes identified
  • Training data provenance documented — sources, licenses, consent
  • Disparate impact tested: performance parity across protected groups measured
  • Red-team exercise conducted: adversarial prompts and misuse scenarios explored
  • Users informed when they are interacting with AI (disclosure standard met)
  • Right to opt-out or human review available for consequential AI decisions
  • EU AI Act risk classification assessed (prohibited / high-risk / limited / minimal)
  • Incident response plan defined for AI failures, bias discoveries, or misuse reports

📊 Metrics & Observability

0 / 7 complete
  • North star metric defined and instrumented in analytics
  • AI-specific metrics live: override rate, correction rate, AI adoption depth
  • Quality monitoring: LLM-judge score or human sampling cadence set up
  • Drift detection: data drift and model drift alerts configured
  • Cost monitoring: cost-per-request tracked with budget alert thresholds
  • Latency SLA set and p95 latency tracked in production
  • Weekly review cadence scheduled for AI quality and product metrics

📄 AI Feature PRD Template

Header

Feature Name:
[Name of the AI feature]
PM Owner:
[Name]
ML Lead:
[Name]
Status:
Draft / In Review / Approved
Target Launch:
[Quarter]

Problem Statement

User Problem:
Who has this problem, when, and what is the cost of not solving it?
Business Goal:
What metric does this move? By how much?
AI Rationale:
Why is AI the right approach? What non-AI alternatives were considered?

ML Problem Formulation

Task Type:
Classification / Generation / Retrieval / Regression / Ranking
Input:
[What goes in — data type, modality, source]
Output:
[What comes out — format, confidence, constraints]
Training Signal:
[Where does the labeling / feedback come from?]

Success Criteria

Model Metric:
e.g. Precision ≥ 0.85, Recall ≥ 0.80, Latency p95 ≤ 400ms
Product Metric:
e.g. AI task completion rate ≥ 70%, override rate ≤ 15%
Business Metric:
e.g. +5% retention, -20% support tickets

Failure Modes & Mitigations

Known Failures:
List top 5 failure modes with severity and frequency estimates
Human Fallback:
What happens when confidence < threshold?
Edge Cases:
OOD inputs, adversarial inputs, language/locale edge cases

Data Requirements

Training Data:
Source, size, quality, bias considerations
Labeling Plan:
Who labels, guidelines, inter-annotator agreement target
Privacy & Consent:
Data retention, PII handling, user consent mechanism

[Single metric to determine winner/loser]

[Metrics that must not regress — e.g. latency, error rate]
Variants:
Control: [current state] / Treatment: [change being tested]
Target Segment:
[Who is in scope — eligibility criteria]
Traffic Split:
e.g. 50/50 or 90/10 for high-risk AI features
Sample Size:
[Calculated N per arm for target MDE]
Run Duration:
[Minimum weeks — must cover novelty period]
Decision Rule:
Ship if p-value < 0.05 AND primary metric lifts ≥ MDE AND all guardrails green
